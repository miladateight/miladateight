Project story

One platform, many layers, clear boundaries.

The project joined corporate website delivery and corporate email into a maintainable operational model. The important work was not a single server install; it was coordinating public DNS, TLS, proxies, tunnels, routing, mail authentication, backups, and recovery so each layer could be validated without exposing production secrets.

My role

End-to-end infrastructure ownership

I designed and deployed the website presentation layer, administered the Linux hosting environment, configured web and mail services, integrated secure remote connectivity, implemented routing policies, validated DNS and TLS behavior, investigated incidents, and documented recovery procedures.

Engineering scope

What was implemented

01

Web and hosting platform

Corporate website delivery, Linux hosting, HestiaCP administration, Nginx, HTTPS behavior, service health, ownership, permissions, and deployment validation.

02

Mail platform

Exim SMTP transport, Dovecot mailbox access, Roundcube webmail, TLS-secured user paths, queue inspection, logs, certificate checks, and reachability validation.

03

Hybrid access paths

Internal users reached services through routed local policy, while external users entered through a VPS, HAProxy, WireGuard, MikroTik, and the controlled hosting network.

04

SMTP delivery policy

Sender-domain based Exim routing selected either direct delivery or secure external relay transport, keeping delivery behavior aligned with SPF, DKIM, DMARC, MX, PTR, and TLS.

05

Network controls

MikroTik firewall, NAT, policy routes, controlled public exposure, encrypted transport, route verification, HAProxy backend checks, and post-change network validation.

06

Operations and recovery

Backup planning, health checks, restore validation, log review, troubleshooting notes, and a documented recovery case for a hosting control panel dependency failure.

Traffic paths

How requests move through the system

Website flow

Public web delivery

DNS -> entry/proxy -> Nginx -> website content

The website layer is delivered through controlled hosting and proxy behavior, with HTTPS, availability, filesystem changes, and service status validated after operational changes.

External mail access

Public mail path through a protected edge

External user -> VPS -> HAProxy -> WireGuard -> MikroTik -> mail services

External users do not directly define the core trust boundary. The VPS is an entry layer, WireGuard carries selected traffic, and MikroTik policy controls what reaches the internal hosting environment.

Internal access

Internal users use the local routed path

Internal user -> MikroTik -> Exim / Dovecot / Roundcube / web

Internal access reduces dependence on the public entry layer while still keeping access policy, service validation, and operational visibility centralized.

Outbound SMTP

Domain policy chooses the route

sender domain -> Exim policy -> direct or relayed transport

Each hosted sender domain can use the delivery route that matches its operational requirements instead of forcing one global outbound path for all mail.

Validation and recovery

Built for operation, not just installation.

Validation model

Changes were checked across the full chain

Validation covered website availability, HTTP to HTTPS behavior, certificate validity, DNS records, SMTP and IMAP reachability, webmail availability, Exim queues, Dovecot health, WireGuard peer state, HAProxy backend reachability, MikroTik routing counters, and restore checks.

Incident recovery

A dependency failure was recovered without treating everything as one blob

When hosting control panel login returned HTTP 500 after a filesystem-related change, recovery separated hosted website content from control-panel application files, restored or validated the dependency layer, checked service health, and documented preventive controls.

Security and privacy boundary

The repository explains decisions, not secrets.

The public portfolio is intentionally restrained. It demonstrates engineering judgment while excluding anything that could identify or reproduce the production environment.
